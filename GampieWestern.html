About Gampie Western

I came up with an idea that had as inspiration the old arcade game "pong" but then western style which cowboys shooting a bullet and deflecting them. Others came with the idea to use ping pong bats to reflect them, I personally thought that was a little bit odd, because realistically you can never deflect a bullet with a ball. But it's a game after all so I agreed, but to add a little bit of spice I came up the idea to add some abilities/power-ups that should spawn random across the map and whenever the bullet collides with the power-up you get the power-up. Further more the name Gampie is from a small company that develops games, it has two 'owners' and I am one of them, so I gave this the name Gampie Western. This game is made in the programming language C#.

My contribution

Well I ofcourse contributed a lot because I created this on my own. Though I've gotten a bit of help from others because when I first made it, I was quiet new to C#. So I had to figure a lot of things out myself. But the art I've been sent was to be honest pretty nice aswell, yet I wasn't able to put all of the art/animations in the game because there were still a few bugs left to fix which were also sort of 'hard' to fix. For example the ability SixShooter was by far the hardest ability to code and also have to work it properly. Since it was such a hard ability to code, with my knowledge, I had to ask a certain person who is specialized in C# and Unity. And together we were able to make the ability just the way it was supposed to work.

Conclusion

Well, my conclusion is that I find it very entertaining to create games with or without people obviously with other developers is more fun but doing it alone (for me) work better because then you cannot ask for help for every problem you run into. So you have to come up with idea's yourself to fix it, and by doing that you learn more then when you ask for help everytime. But I'm still proud of the game and it works pretty much as I had in mind, it was very amazing to see the game in the end because everything worked fine, without any errors.




Nederlands
Over Gampie Western

Ik kwam met een idee dat als inspiratie het oude arcadespel "pong" had, maar dan westernstijl waarbij cowboys een kogel afschieten en ze afbuigen. Anderen kwamen met het idee om ping pong batjes te gebruiken om ze te reflecteren, ik persoonlijk vond dat een beetje een beetje vreemd, want realistisch gezien kun je nooit een kogel afbuigen met een ping pong batjes. Maar het is tenslotte een spel dus ik ging akkoord, maar om het een beetje spannender te maken kwam ik op het idee om een aantal abilities/power-ups toe te voegen die willekeurig over de kaart moeten spawnen en wanneer de kogel tegen de power-up botst krijg je de power-up. Verder is de naam Gampie van een klein bedrijfje dat games ontwikkelt, het heeft twee 'eigenaren' en ik ben een van hen, dus gaf ik dit de naam Gampie Western gegeven.

Mijn bijdrage

Nou, ik heb natuurlijk veel bijgedragen omdat ik dit zelf heb gemaakt. Hoewel ik wel een beetje hulp van anderen, want toen ik het voor het eerst maakte, was ik nog vrij nieuw in C#. Dus heb ik veel dingen zelf moeten uitzoeken. Maar de art die ik heb gekregen was om eerlijk te zijn best mooi, maar ik was niet in staat om alle art/animaties in het spel te zetten omdat er nog een paar bugs moesten worden opgelost die ook een soort van 'moeilijk' waren om op te lossen. Bijvoorbeeld de ability SixShooter was veruit de moeilijkste ability om te coderen en ook nog goed te laten werken. Omdat het zo'n moeilijke ability was om te coderen, met zover ik wist, moest ik een bepaald persoon vragen die gespecialiseerd is in C# en Unity. En samen waren we in staat om de mogelijkheid te maken precies zoals het zou moeten werken.

Conclusie

Mijn conclusie is dat ik het erg leuk vind om spellen te maken met of zonder mensen met andere developers is natuurlijk leuker maar het alleen werken werkt (voor mij) beter omdat dan kan je niet voor elk probleem waar je tegenaan loopt om hulp vragen. Dus moet je zelf met ideeën komen zelf om het op te lossen, en door dat te doen leer je meer dan wanneer je elke keer om hulp vraagt. Maar ik ben nog steeds trots op het spel en het werkt ongeveer zoals ik het in gedachten had, het was erg verbazingwekkend om het spel te zien op het einde omdat alles goed werkte, zonder fouten.
